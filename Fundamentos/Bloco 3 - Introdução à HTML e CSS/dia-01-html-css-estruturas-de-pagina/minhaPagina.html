Unix e Shell

ls - Listar pastas e arquivos contidos no diretório atual.

pwd - Mostrar o caminho completo do diretório atual.

mkdir - Make Directory - Criar um novo diretório.

cd - Change Directory - Mudar de diretório.

. - Refere-se ao diretório que o usuario está.

.. - Refere-se ao diretório pai, anterior.

clear - Limpa os comandos do terminal.

? - Capaz de substituir um caracter na busca de arquivos e diretórios.

Git e GitHub

O Git é um sistema de controle de versão que permite o trabalho de mais de uma pessoa em um mesmo arquivo ou projeto e que provê ferramentas parroblemas que possam acontecer nesse tipo de situação.

O GitHub é uma plataforma que utiliza o sistema do Git para o controle das versões de algum arquivo ou projeto.

Principais comandos do Git:

-git init - inicia o repositório git local, em nosso computador;

-git status - ver o status do git;

-git add - para adicionar os arquivos - git add 'nome-do-arquivo' ou git add .

O git add . adiciona todos os itens que estão dentro do diretório atual, incluindo subdiretórios e arquivos.

git commit -m "qual a ação dos arquivos, o que esse arquivo irá fazer no projeto"

O git commit cria uma revisão com um número e comentário que será visto por todos.

git log - mostra todas as informações de commit's feitos para aquele projeto.

git rm nome-do-arquivo - Apagar o arquivo que não será utilizado. (Após a remoção do arquivo, é importante fazer um commit para manter atualizada as alterações feitas no Git)

git log --diff-filter=D -- summary - Este comando irá trazer todos os commits que foram apagados, para que possamos retornar o arquivo se necessário.

git checkout numero-do-commit-que-deseja-retornar~1 - Com este comando, o arquivo será 'ressucitado', volta a existir.

git ignore - Serve para 'ignorar' arquivos que não se deseja adicionar para o git.

É necessário criar o arquivo .gitignore com o touch e editar o arquivo, inserindo nele o arquivo ou até mesmo a extensão de arquivos, utilizando o *. Ex.: *.txt.

Após a criação e edição do arquivo, é necessário adicionar o arquivo .gitignore e fazer um commit do mesmo.

É possivel ignorar um diretório inteiro se quiser, editando o arquivo .gitignore colocando o nome do diretório seguido de /.

git remote origin e link criado pelo GitHub - Esse comando cria um vinculo entre o repositório local e o repositório remoto.

git push -u origin master - enviar o arquivo para o GitHub. (A primeira vez que for enviado para o GitHub, utiliza-se o -u. Das outras vezes não é mais necessário).

git clone 'link do repositório no GitHub' - clona todo o repositório do GitHub para o computador.

git fetch - Verificar se existe alguma alteração com minha Branch que está no repositório remoto.

git pull - Verifica se tem uma alteração e faz um merge da branch remota para a branch local.



Internet

Internet - Entendendo como ela funciona

Na aula de hoje, entendemos um pouco mais como a Internet funciona.

Há muitos processos por trás do momento em que digitamos o endereço do site e clicamos em Enter.

Inicialmente, no momento em que pesquisamos pelo site(www.google.com), uma solicitação sai do nosso cliente(Navegador) e vai até um servidor DNS (Domain Name System), que por sua vez irá pesquisar pelo nome do site e encontrar seu respectivo IP (Internet Protocol). Sim, todos os sites possuem um número de IP, mas seria impossivel decorar todos.

Com a resposta do DNS é que será efetuada uma requisição aos servidores dos sites, por meio do protocolo HTTP (Hiper-Text Transfer Protocol), protocolo de comunicação entre servidores e cliente.

Ao chegar a informação até o servidor dos sites, este fará uma pesquisa em seu banco de dados e retornará a informação para o cliente. Toda essa resposta enviada pelo servidor será renderizada pelo cliente e criará a página do site pesquisado.

Esse ciclo só acontece com o funcionamento do protocolo de comunicação TCP/IP (Transmission Control Protocol/Internet Protocol), utilizado para transmitir os dados na rede. Outro quesito que não pode faltar é o funcionamento adequado do nosso ISP:ISP (Internet Service Provider), que é nosso provedor de internet responsável por fornecer o acesso a Internet.

Ainda há outros mecanismos usados no tráfego de informações para que os servidores possam trabalhar com milhões de requisições, como por exemplo o balanceador de carga. Se trata de um algoritmo responsável por distribuir as requisições para o servidor.
